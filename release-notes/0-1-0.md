# Version 0.1.0

This version delivers a complete Atlas toolkit, a restructured command interface, and the first full documentation release for **WoT Utils**.

> **Breaking change:** The CLI binary has been renamed. Replace all calls to `wg-fetcher` with `pie-wot`. Command paths have also changed — see the **Improved** section for the full mapping.

## New

### Atlas: Inspect an atlas sprite manifest

Your Commander can now list every texture name stored in an atlas XML file before extracting anything. Run `atlas inspect` against any `.xml` manifest to receive an alphabetically sorted inventory of available sprites — map icons, vehicle silhouettes, UI elements, and more.

```bash
npm start -- atlas inspect maps/vehicles.xml
```

### Atlas: Extract all textures at once

`atlas extract` pulls every sprite out of an atlas in a single operation, writing one PNG file per texture into a destination directory of your choice. Supply either a base path (the tool derives the `.png` and `.xml` automatically) or explicit `--image` and `--map` paths for full control.

```bash
npm start -- atlas extract --from maps/vehicles --to output/vehicles/
```

### Atlas: Pack a directory of PNGs into a new atlas

`atlas pack` assembles a folder of PNG files into a fresh texture atlas, producing a paired `.png` image and `.xml` sprite manifest. Use `--padding` to add transparent gutters between sprites, and `--max-width` to enforce a maximum atlas dimension.

```bash
npm start -- atlas pack --from sprites/ --to output/custom_atlas --padding 2
```

### Background image generation

Run `pie-wot icon dump-background` to generate per-type background images — Heavy Tanks, Light Tanks, Medium Tanks, Tank Destroyers, and SPGs — in PogS style. Each image is written as a PNG to your current directory, ready to composite into a custom icon set.

```bash
pie-wot icon dump-background
```

### Full command reference documentation

Complete documentation for all command groups — `vehicle`, `atlas`, `font`, and `cache` — is now available, including step-by-step guides, argument tables, and worked examples for every command.

## Improved

### Commands reorganized into focused groups

All commands are now grouped under four top-level namespaces, replacing the previous flat command list:

| Old command                | New command                   |
|----------------------------|-------------------------------|
| `wg-fetcher list-vehicles` | `pie-wot vehicle list`        |
| `wg-fetcher export`        | `pie-wot vehicle export`      |
| `wg-fetcher fetch-icons`   | `pie-wot vehicle fetch-icons` |
| `wg-fetcher best-config`   | `pie-wot vehicle best-config` |
| `wg-fetcher vehicle-stats` | `pie-wot vehicle stats`       |
| `wg-fetcher chars`         | `pie-wot vehicle chars`       |
| `wg-fetcher pick`          | `pie-wot atlas pick`          |
| `wg-fetcher cache-purge`   | `pie-wot cache purge`         |

Run `pie-wot --help` or `pie-wot <group> --help` to explore all available options.

### Toolkit renamed to WoT Utils

The project is now published as **WoT Utils** and the CLI binary is `pie-wot`. The previous binary name `wg-fetcher` is no longer active. Update any scripts or aliases that invoke the old name.

### `atlas pick` error message now guides your next step

When `atlas pick` cannot locate the requested texture name, the error message now directs you to run `atlas inspect` on the same manifest to retrieve the full list of available names — eliminating the need to parse the XML manually.

## Fixed

- **CLI works from any directory** — `pie-wot` now resolves its configuration and assets correctly when run from outside the project root (e.g., after `npm link`). Previously, running the toolkit from an unrelated directory caused it to fail to load the `.env` file and the shield overlay asset.
